hexo-写文章(写作篇)

1. 前言

框架搭建完成了。接下来开始在hexo个人博客中写第一篇文章

2. 创建新页面

hexo new <title> title 改为文章标题

可以看到 /source/_posts/ 目录下多了一个对应标题的md文件。在这个文件中就可以使用markdown语法进行编写了。

3. 将新页面更新到仓库

执行 hexo d -g 往远程仓库更新一下博客站点
执行完毕后，刷新博客站点，可以看到新的文章出现了。

hexo-尝试使用（初始化、部署篇）

前言

突然想整一个个人博客试试。在网上看到说hexo好上手，并且很久以前发自己往电脑上装过hexo，所以就拿这个试试。

安装

因为电脑上已经有了，就不写怎么安装了。就是node+npm

更新hexo

首先面临的问题，就是升级hexo。毕竟是两年前安装的版本，还是先升级到最新的好。

  1. 查看现有版本

    可以看到是比较老的版本了。目前的最新版本已经是3.1.0了。更新！

  2. 更新
    重新安装就是更新，在任意目录执行 npm install hexo-cli -g
    结果

    接下来就放心大胆的使用了。

    使用

  3. 新建个目录，并在目录下初始化项目hexo init my-blog。初始化比较慢，可能是因为要从github克隆代码的原因……

  4. 安装依赖。

    初始化完成~

  5. 运行本地环境hexo s

    在浏览器中打开本地地址：http://localhost:4000。就成功看到了默认的博客了！

部署到github上

github 有基于仓库生成静态网站的功能。

  1. 创建仓库 获取网址

    创建时要注意：
    仓库名有固定格式：账户名.github.io。Initialize this repository with a README要勾选上。
    创建成功后，在仓库设置中有一个GitHub Pages设置，展示出来了一个链接，这个链接就是能够访问仓库中的网站的网址了。
  2. 配置Deployment
    在博客项目的根目录有个_config.yml配置文件，修改里面的deploy配置，如下：
  3. 安装github部署插件
    部署到git需要hexo-deployer-git这个插件。在根目录执行：npm install hexo-deployer-git --save
  4. 部署至仓库
    执行远程部署命令hexo d -g, 执行完毕后，打开 1. 创建仓库 获取网址 这一步中拿到的网址，就能看到博客部署成功了。

回顾一下命令

hexo -v 查看hexo版本
npm install hexo-cli -g 安装/更新 hexo
hexo init my-blog 初始化创建项目，‘my-blog’ 随意写
npm i 创建完项目后安装依赖
hexo s 本地运行预发布环境
npm install hexo-deployer-git --save 安装部署到github的插件
hexo d -g 远程部署到仓库
